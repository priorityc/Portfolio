Southend Pier Website — UX Redesign Case Study

Overview

The Southend Pier website is a high‑traffic tourist site with outdated structure, unclear navigation, and inconsistent visual hierarchy. This redesign focuses on clarity, mobile usability, and a modern, task‑based layout.

Problem

Visitors struggle to find essential information such as ticket prices, opening times, and directions. The homepage lacks a clear hero message and the navigation is overloaded, making the site difficult to use on both desktop and mobile.

My Role

I performed a full UX audit, identified usability issues, and created a redesigned homepage concept focusing on hierarchy, accessibility, and mobile-first layout.

UX Audit Findings

  • Overloaded navigation with unclear grouping
  • No primary call‑to‑action on the homepage
  • Inconsistent spacing and typography
  • Poor mobile responsiveness
  • Low contrast text and missing accessibility features

Redesign Goals

  • Create a clear hero section with a single CTA
  • Simplify navigation into task‑based categories
  • Improve mobile layout and readability
  • Introduce consistent spacing and typography
  • Enhance accessibility and contrast

Redesign Concept

The redesigned homepage features a bold hero banner, a simplified navigation structure, and clear pathways for key user tasks such as buying tickets, planning a visit, and viewing events. The layout uses a modern grid system with improved spacing and visual hierarchy.

What I Learned

This project strengthened my ability to evaluate real‑world websites, identify UX issues, and propose practical redesign solutions. It also improved my understanding of accessibility, mobile-first design, and task‑based navigation.
