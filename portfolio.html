Introduction üìò

The Yorvik Parking App is a mobile-first prototype designed to help users register and book longboat parking spaces in the historic city of York. Built with stakeholder collaboration at its core, the app transforms a niche service into a seamless digital experience ‚Äî from registration to QR-based ticketing.

The app was build as functional prototype to help stakeholders visualize their service in action before commiting to full-scale development.

Libralies and frameworks were selected based on stakeholder needs, team familiarity and scalability. Each tool was tested to ensure optimal performance.

The React Native client must:

  • Register a boat captain
  • Select a start and finish date and time
  • Select an available parking berth on a map
  • Request a berthing permit for the selected space for the selected period
  • isplay a berthing permit with a QR code
  • View all previously issued berthing permits

The Web service must:

  • Maintain a list of parking berths, including a description and their latitudes and longitudes
  • Maintain a list of issued berthing permits
  • Provide a list of available parking berths for a given period
  • Create a berthing permit for the given berth and period for a captain, and mark the berth asunavailable
  • Return a list of all berthing permits that a captain has been issued
Tech: React Native, TypeScript, Libralies:react-native-qrcode-svg Tools: Expo Go, Vite, npm, Express.js

Demo Mode Flow üß™

Safely test the checkout using ?demo=true. No real payment is processed. The frontend detects demo mode and sends mock data to the backend, which returns a Stripe test session.

How?

  • Persists across navigation and reloads using localStorage
  • Sends mock product data to the backend
  • Returns a Stripe test session with no real payment required
  • Visually mirrors the real checkout flow for authenticity
